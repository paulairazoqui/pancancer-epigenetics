A lineage-aware computational oncology framework designed to identify recurrent, co-regulated epigenetic-transcriptomic programs across diverse malignancies and systematically evaluate their statistical associations with resistance-like pharmacogenomic contexts, candidate functional dependencies, and perturbational configurations.

---

## Project Overview

This repository integrates high-throughput, multi-omic, pharmacogenomic, functional dependency, and perturbational transcriptomic datasets to map latent biological states across heterogeneous tumor and *in vitro* models. 

### Core Data Integration
* **Primary Tumors:** The Cancer Genome Atlas (TCGA)
* **Functional Genomics:** Cancer Dependency Map (DepMap) / Cancer Cell Line Encyclopedia (CCLE)
* **Pharmacogenomics:** GDSC, CTRP, and PRISM
* **Perturbational Profiles:** LINCS L1000 / Connectivity Map (CMap)

The underlying framework is engineered around the principles of strict biological interpretability, rigorous data leakage prevention, and conservative scientific framing.

---

## Out-of-Scope Declarations

To maintain scientific validity and manage expectations regarding purely computational associations, this framework is **not** designed to:
* Predict clinical patient outcomes or survival metrics.
* Infer definitive causal or mechanistic biological pathways.
* Reconstruct longitudinal, adaptive, or clonal drug-resistance trajectories.
* Validate or claim translational therapeutic efficacy or clinical biomarkers.

> [!NOTE]
> All outputs generated by this pipeline represent computational associations and candidate hypotheses that require prospective downstream wet-lab validation.

## Repository Structure

```text
├── data/                  # Multi-omic and screening data tiers
│   ├── raw/               # Immutable raw source datasets
│   ├── interim/           # Partially processed or harmonized data
│   └── processed/         # Final analytical matrices and program definitions
├── docs/                  # Detailed architectural and framework documentation
├── envs/                  # Conda/Mamba environment definitions for reproducibility
├── notebooks/             # Exploratory data analysis (EDA) and prototyping
└── src/                   # Production-grade, version-controlled source code
```

## Documentation
Comprehensive conceptual and technical documentation is organized within the docs/ directory. For deep dives into the pipeline's logic, please consult:
- PROJECT_DIRECTION.md — Strategic biological overview and long-term research goals.
- PROJECT_ARCHITECTURE.md — Detailed breakdown of the four analytical layers and pan-cancer confounding controls.
- DATA_SOURCES.md — Detailed curation, preprocessing, and standardization protocols for each dataset.

## Reproducibility & Rigor
**Data Immutability:** Files within data/raw/ must remain completely unmodified.

**Deterministic Pipeline:** All downstream outputs in data/interim/ and data/processed/ are fully reproducible from the raw source files using the version-controlled workflows located in src/ and the environment configurations in envs/.
